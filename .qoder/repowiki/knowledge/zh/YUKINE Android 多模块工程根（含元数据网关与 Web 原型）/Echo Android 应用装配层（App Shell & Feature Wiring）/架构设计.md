本模块是单 APK 的顶层装配层，不实现业务逻辑，仅把 `:feature:*` 与 `:core:*` 模块拼成可运行的 App。
- 入口：`EchoApplication`（`@HiltAndroidApp`）在 `onCreate` 中启动后台任务调度器并安装 CrashLogger；`MainActivity`（`singleTop` + `echo-next://streaming-auth` 回调）通过 `MainActivityComposition` 按顺序初始化 settings → onboarding → navigation → playback → streaming，并在生命周期中透传可见性。
- DI 分层：`di/` 下每个 `*Module.kt` 以 `@InstallIn(SingletonComponent::class)` 暴露接口到 `SingletonComponent`，由上层 `MainActivityDependencies` 组合为 `MainActivityFeatureBindings`，再被 `MainActivity` 消费，形成「模块提供接口 → 装配层组装 → Activity 使用」的单向依赖。
- 播放服务：`playback/EchoPlaybackService.java` 继承 `MediaLibraryService`，持有 `PlaybackServiceRuntime`（由 `PlaybackServiceRuntimeFactory` 创建），对外暴露 `PlaybackServiceHostPort` Binder 供 UI 进程通信；通知、前台、Action 分发均委托给 runtime。
- 原生桥接：`src/main/cpp/` 通过 CMakeLists.txt + Android.mk 编译 `echo_chromaprint_jni.cpp`，NDK ABI 限定 arm64-v8a/x86_64，JNI 暴露给 `fingerprint/ChromaprintNative.kt`。
- 构建产物：Room schema 输出到 `schemas/`，lint 基线在 `lint-baseline.xml`，签名与版本信息通过 Gradle property / 环境变量注入（`ECHO_RELEASE_*`、`YUKINE_QQ_APP_ID`、`ECHO_METADATA_GATEWAY_URL`）。
