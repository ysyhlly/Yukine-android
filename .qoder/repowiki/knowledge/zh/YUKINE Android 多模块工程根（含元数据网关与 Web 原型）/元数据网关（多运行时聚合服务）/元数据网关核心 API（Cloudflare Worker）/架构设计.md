模块由三层组成：
- 入口层 `index.ts`：导出符合 Cloudflare Workers 规范的默认对象，实现 `fetch(request, env)`，将 `Request` 适配为内部 `GatewayRequest`，注入 `JsonFetchTransport` 与运行时环境后交给核心处理。
- 核心路由层 `core.ts`：`handleGatewayRequest` 按 pathname 分发到 `recordings`、`artists`、`lyrics` 三个处理器；每个处理器负责参数校验（bounded/uuid/integer/escapeLucene）、调用上游、合并证据、去重排序并返回 `GatewayResult`。
- 类型契约层 `types.ts`：定义 `UpstreamTransport` 接口、`UpstreamJsonResult` 联合类型、`GatewayEnvironment`、`GatewayContext`、`GatewayResult`、`RequestTrace` 等，使业务逻辑与具体 HTTP 传输解耦。

依赖方向单向：`index.ts → core.ts → types.ts`；所有外部 I/O 通过 `context.transport.getJson` 抽象，便于在 Node 环境下用不同 transport 测试。上游常量集中在文件顶部（MB/ITUNES/ACOUSTID/WIKIDATA/LRCLIB），并通过 `upstream()` 统一包装，自动记录 trace 与可达性统计，当所有上游均不可达时回退为 502 upstream_failure。
