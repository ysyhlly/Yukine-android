根目录作为多语言多运行时聚合层：
- settings.gradle 声明所有 Android 子模块（:app、:core:*、:feature:*），并通过 dependencyResolutionManagement.repositoriesMode = FAIL_ON_PROJECT_REPOS 强制各子模块集中从 google/mavenCentral 拉取依赖，避免各自维护仓库源。
- build.gradle 仅通过 alias(libs.plugins.*) apply false 在根级别注册插件版本，由子模块按需引入，保证插件版本全局一致。
- 非 Android 子项目以目录形式存在：metadata_gateway/（Cloudflare Worker + Node.js 本地代理）与 web_prototype/（Vite+React），它们不纳入 Gradle 构建，但被 scripts_and_artifacts/ 中的 PowerShell 脚本在发布门禁中串联调用；docs_and_audits/ 存放跨模块的架构说明与审计归档。
- 根 README 是唯一的对外入口，统一描述应用能力、环境要求（JDK 17 / SDK 35 / CMake 3.22.1）、构建命令与文档索引，子模块不再重复声明这些契约。
