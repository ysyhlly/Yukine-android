本模块作为 `:core:*` 与 `:feature:*` 的容器，不实现业务逻辑，仅做运行时装配：
- 进程入口：`EchoApplication`（`@HiltAndroidApp`）在 `onCreate` 中执行备份恢复、CrashLogger 安装及多个 WorkManager 调度器；`MainActivity` 通过 `singleTop` + `echo-next://streaming-auth` 回调接收 WebAuth。
- DI 分层：`di/` 下每个 `*Module.kt` 以 `@Module @InstallIn(SingletonComponent::class) object XxxModule` 暴露接口到 `SingletonComponent`，由 `MainActivityDependencies` 组合为 `MainActivityFeatureBindings`，再被 `MainActivityComposition` 按 feature 顺序组装，形成「模块提供 → 装配层组合 → Activity 消费」的单向依赖。
- Feature 装配：`MainActivityComposition.create()` 依次构建 Settings → Platform → Navigation → Streaming → Library → Playback → Network → Onboarding 等 `*FeatureBinding`，并通过 `bindConnection/bindFeatures/bindUi/bindDialogs` 等方法互相桥接。
- 播放服务：`playback/EchoPlaybackService.java` 继承 `MediaLibraryService`，持有 `PlaybackServiceRuntime`（由 `PlaybackServiceRuntimeFactory` 创建），对外暴露 `PlaybackServiceHostPort` Binder 供 UI 进程通信；通知、前台、Action 分发均委托给 runtime。
- 原生桥接：`src/main/cpp/` 通过 CMakeLists.txt + Android.mk 编译 vendor/chromaprint 与 `echo_chromaprint_jni.cpp`，NDK ABI 限定 arm64-v8a/x86_64，JNI 暴露给 `fingerprint/ChromaprintNative.kt`。
- 构建产物：Room schema 输出到 `schemas/`，lint 基线在 `lint-baseline.xml`，签名与版本信息通过 Gradle property / 环境变量注入。
