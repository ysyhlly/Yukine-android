- 对外部 JSON 响应一律通过 `object()` / `array()` / `string()` / `number()` 安全访问器读取，禁止直接下标或链式属性访问，避免上游字段缺失导致崩溃。
- 入参统一经 `bounded`、`uuid`、`integer`、`escapeLucene` 等校验函数清洗后再进入上游查询，非法值回退为空或默认边界。
- 每次上游调用走 `upstream()` 包装，统一记录 `trace.upstream` 与 `summary.attempted/reachable`，并在全部失败时返回 `upstreamFailure` 502。
- 结果通过 `result(body, status, trace, maxAge?)` 工厂构造，固定设置 `Content-Type`、`Cache-Control`、`X-Content-Type-Options` 响应头。
- 多上游聚合采用“证据列表”模式：各来源产出 `RecordingEvidence` 数组，最后经 `dedupeRecordings` 按 score 降序去重截断。
