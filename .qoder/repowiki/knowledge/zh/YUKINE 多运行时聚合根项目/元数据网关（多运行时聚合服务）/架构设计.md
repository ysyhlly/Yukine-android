模块按运行时拆分：`src/index.ts` 导出 Cloudflare Worker `fetch` 入口，`src/node/server.ts` 提供 Node 版 HTTP 服务器，两者都通过 `handleGatewayRequest(request, context)` 进入同一套核心路由与编排逻辑（`core.ts`），由 `transport.ts` 中的 `JsonFetchTransport` 抽象上游 HTTP 调用，并通过 `UpstreamTransport` 接口注入可替换实现（Worker 使用内置 `fetch` + Cloudflare Cache，Node 可通过 `sqlite-cache.ts` 持久化缓存）。`types.ts` 定义跨运行时的共享契约（`GatewayRequest`/`GatewayContext`/`GatewayResult`/`UpstreamTransport`），使核心业务不感知底层运行时差异；`wrangler.toml` 与 `tsconfig.worker.json` 约束 Worker 构建，`tsconfig.node.json` 与 `compose.yaml` 支撑本地 Docker 调试。
