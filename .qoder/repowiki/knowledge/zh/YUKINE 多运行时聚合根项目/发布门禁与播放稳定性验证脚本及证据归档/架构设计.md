模块由两类目录组成：
- `scripts/`：三个独立的 PowerShell 入口，均使用 `param()` 声明开关并通过 `Resolve-Adb` 从项目本地 `android-sdk/platform-tools/adb.exe`、用户 SDK 或 PATH 解析 adb。
  - `p0-stability-gate.ps1`：P0 门禁总控，按顺序调用 Gradle 运行 `StreamingViewModelTest`、`EchoDatabaseHelperTest`、`:feature:playback:testDebugUnitTest`，可选 `assembleDebug`，并在有设备时触发 `playback-stability-smoke.ps1`；所有步骤通过 `Invoke-Checked` 包装并以 Markdown 表格写入 `app/build/p0-stability-gate/<timestamp>.md`。
  - `playback-stability-smoke.ps1`：播放稳定性冒烟，负责 assemble/debug APK 安装、按 Android 版本动态授予 `READ_MEDIA_AUDIO`/`POST_NOTIFICATIONS` 或 `READ_EXTERNAL_STORAGE`、启动 MainActivity、采集截图与 dumpsys（activity/media_session/notification/audio）、拉取 logcat 并断言无 FATAL，同时生成 `matrix-results.md` 模板供人工填写回放矩阵结果。
  - `verify-release.ps1`：Release 签发前校验，可并行执行 assemble/lint/bundle（debug + release），支持 `-Connected` 走 `connectedDebugAndroidTest` 或手动指定 `-DeviceSerial` 安装 debug+androidTest APK 并运行 instrumentation；`-ReleaseSmoke` 分支通过环境变量 `ECHO_RELEASE_*` 或 `-CreateSmokeKeystore` 临时 keytool 生成 keystore 完成签名，安装后断言 MainActivity 处于 Resumed 且 logcat 无致命崩溃。
- `artifacts/`：只读证据仓库，按场景分目录存放截图（`.png`）、Room 数据库备份（`echo_next.db` / `.db-journal`）、录屏元数据（`source-ingestor-library-window.xml`）以及设备预装/后同步阶段的数据库快照，命名以设备序列号或时间戳为前缀，便于回溯。
