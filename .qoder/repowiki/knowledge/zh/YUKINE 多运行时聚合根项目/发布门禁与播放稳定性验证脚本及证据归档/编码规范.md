- 每个脚本均以 `[switch]/[string]` 形式的 `param()` 暴露可选参数，并通过 `$ErrorActionPreference = 'Stop'` 与 `Set-StrictMode -Version Latest` 开启严格模式。
- 统一封装 `Invoke-Checked` 函数包装外部命令，失败时抛出包含标题和退出码的异常，使主流程保持线性 try/catch 风格。
- adb 路径解析遵循固定优先级：项目本地 `android-sdk/platform-tools/adb.exe` → `%LOCALAPPDATA%/Android/Sdk/platform-tools/adb.exe` → PATH 中的 `adb.exe`，并通过 `Invoke-Adb` / `Invoke-AdbCapture` / `Invoke-AdbAllowFailure` 三件套根据是否指定 `-DeviceSerial` 自动注入 `-s <serial>`。
- 权限授予逻辑按 `ro.build.version.sdk >= 33` 分支选择 `READ_MEDIA_AUDIO` + `POST_NOTIFICATIONS` 或旧版 `READ_EXTERNAL_STORAGE`，避免在 API 33+ 设备上因权限变更导致安装后无法读取媒体。
- 证据输出采用“时间戳目录 + 编号前缀”约定（如 `yyyyMMdd-HHmmss`、`01-launch.png`、`01-logcat-tail.txt`），并在目录内生成 `README.md` 与 `matrix-results.md` 模板说明证据含义。
- 对外部工具（adb/keytool/gradle）的调用一律通过相对 `ProjectRoot` 拼接路径，不假设工作目录，脚本内部用 `Push-Location $ProjectRoot` / `Pop-Location` 保证幂等。
