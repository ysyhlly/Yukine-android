## 真实存在的问题（按影响排序）

### P0 - 阻碍核心场景

#### P0-1　在线音乐登录入口不可发现（体验评分 4/10）

**症状**：
- "音源与网络"页面只显示 WebDAV、网关配置等高级选项
- 完全找不到"登录网易云"或"登录QQ音乐"的按钮
- 搜索页显示"在线音乐 0首"，但不知道怎么连接
- 首页"连接流媒体账号"卡片点击后效果不明显

**根因（已确认）**：
- 登录能力**完整存在**：`StreamingWebAuthActivity` + `StreamingAuthLauncher`（[MainActivity.java:243](../app/src/main/java/app/yukine/MainActivity.java#L243) 已接线）
- 本地直连也存在：`LocalNeteaseStreamingClient`、`LocalQqMusicStreamingClient`
- 问题是**信息架构**：技术配置（网关IP/端口）淹没了用户真正需要的"登录"入口

**改法**：
1. **流媒体hub页顶部增加醒目按钮**："登录网易云音乐" / "登录QQ音乐"，直接触发 `StreamingAuthLauncher`
2. **网关配置收进"高级设置"折叠区**，默认隐藏
3. **首页"连接流媒体"卡片直达登录选择**，而非网关配置页

**实施前需定位**：
- `NETWORK_STREAMING_HUB` 路由对应的 Compose screen（查 `MainRoutes` + navigation 映射）
- 确认该页面的现有结构和登录触发方式

**涉及文件**：
- 流媒体 hub screen（待定位）
- [MainActivity.java](../app/src/main/java/app/yukine/MainActivity.java)（导航目标）
- [AppLanguage.java](../app/src/main/java/app/yukine/AppLanguage.java)（文案本地化）

**风险**：中。需重排 UI 结构，但不动核心逻辑。

### P1 - 新用户体验

#### P1-1　新手引导缺失

**症状**：
- 首次启动没有引导，用户需要自己摸索
- 各功能入口位置不明显（歌单、队列、设置等）

**建议**：
- 首次启动增加简单的功能引导（可选的 overlay 提示或欢迎页）
- 标注关键功能入口（首次点击时显示功能说明）

**优先级**：P1（影响上手体验，但不阻碍功能使用）

---

#### P1-2　歌单功能不明显

**症状**：

**需定位**：

**改法**：
- 如果功能存在，调整入口位置使其更明显
- 如果缺失，按 P1 优先级补充

### P2 - 打磨

#### P2-1　"音源与网络"页面对普通用户不友好

**症状**：
- WebDAV、网关、流媒体音质等概念太专业
- 普通用户看不懂

**建议**：
- 分为"基础设置"和"高级设置"两个区域
- 基础设置只显示登录、音质选择等常用项
- 高级设置折叠显示 WebDAV、网关等专业配置

## 与 MVVM 迁移目标的对齐说明（2026-06-25）

这份体验审查现在同时作为迁移优先级输入使用，后续接手时按下面原则理解：

1. 不把体验问题和架构迁移割裂看待。
   在线登录入口、播放状态中转、下载与缓存响应、状态消息路径，优先通过明确 owner 和缩短 `MainActivity` 链路解决。

2. 不把审查里的完成度百分比当作验收事实。
   迁移是否完成，仍以 `docs/MVVM_MIGRATION_HANDOFF.md` 的目标、当前工作树、编译/测试/设备验证为准。

3. 当前优先级顺序保持：
   先收缩 `MainActivity` 和壳层状态同步，再统一播放状态源，再推进曲库/歌单和流媒体拆分。

4. 与本报告直接相关的迁移关注点：
   流媒体登录入口可发现性、播放启动路径卡顿、缓存/下载路径、状态反馈统一出口、首屏和播放页状态中转。
